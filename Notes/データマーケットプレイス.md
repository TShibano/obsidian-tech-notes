## 概要

データマーケットプレイス（Data Marketplace）は，データプロデューサーとデータコンシューマーがデータを取引・共有するオンラインプラットフォームである．データセットの公開・探索・購入・配信を一元的に管理し，組織内外のデータ流通を促進する．2020年代初頭に登場したプロダクトで，これから伸びると予想される．

## 詳細

### 3つのコンポーネント

```
データプロデューサー
    ↓ データアセットを公開
[データマーケットプレイス]
    ↓ 探索・アクセスリクエスト
データコンシューマー
    ↑ リクエスト承認・配信
データスチュワード（管理者）
```

1. **プロデューサー**: データセットを作成・公開・管理
2. **コンシューマー**: カタログを探索してデータにアクセスをリクエスト
3. **スチュワード**: アクセス承認・ガバナンス・品質管理を担当

### 内部型と外部型

| 種別 | 説明 | 例 |
|------|------|----|
| 内部型（Enterprise） | 組織内部でのデータ共有・セルフサービス | Informatica Cloud Data Marketplace |
| 外部型（Commercial） | 第三者データの売買プラットフォーム | AWS Data Exchange，Snowflake Marketplace |

### データメッシュとの関係

[[データメッシュ]] アーキテクチャにおけるデータプロダクトの配信インフラとしてデータマーケットプレイスが機能する．ドメインチームがデータプロダクトをマーケットプレイスに登録し，他ドメインがセルフサービスで利用する．

### データカタログとの違い

| 項目 | データカタログ | データマーケットプレイス |
|------|--------------|----------------------|
| 主目的 | メタデータ管理・データ発見 | データ流通・取引 |
| アクセス制御 | メタデータベース | 承認フロー・課金対応 |
| 対象 | 主に内部データ資産 | 内部＋外部データ |
| 機能 | 検索・分類・リネージ | 購入・契約・配信 |

### ユースケース

- **気象データ活用**: 小売業が外部気象データを購入し来客数・売上予測に活用
- **在庫・物流最適化**: 外部サプライチェーンデータとの連携
- **AI/ML 学習データ**: 機械学習用データセットの調達
- **社内データ民主化**: データエンジニアが整備したデータセットをビジネス部門へ展開

### 主要なデータマーケットプレイス

- **AWS Data Exchange**: サードパーティデータの購読サービス
- **Snowflake Marketplace**: Snowflakeエコシステム内でのデータ共有
- **Azure Data Marketplace**: Microsoftエコシステムでのデータ流通
- **Informatica Cloud Data Marketplace**: エンタープライズ向け社内データ流通

## ポイント

- データガバナンスとセキュリティが成功の鍵（誰が何にアクセスできるか）
- [[データカタログ]] との統合により，データ発見から取得までをシームレスに
- クラウドプラットフォームがマーケットプレイス機能を内包するトレンドが加速
- データプロダクト思考（[[データメッシュ]]）と相性が良い
