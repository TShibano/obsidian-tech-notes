## 概要

Redox OS は Rust で書かれたマイクロカーネルベースの Unix 互換オペレーティングシステム。「OS をできる限り安全にする」というビジョンのもと、カーネルにファイルシステムやドライバを含まない純粋なマイクロカーネル設計と、スキームベースの IPC（プロセス間通信）を採用している。

### マイクロカーネル設計

従来のモノリシックカーネル（Linux 等）がファイルシステム・ドライバ・ネットワークスタックをカーネル空間に含むのに対し、Redox OS のマイクロカーネルはメモリ管理・プロセス管理・IPC という最小限の機能のみをカーネル空間に実装する。

ファイルシステム・ドライバ・ネットワークデーモンはすべてユーザー空間で動作し、クラッシュしてもシステム全体が落ちない。

```
┌─────────────────────────────────────┐
│           ユーザー空間               │
│  ┌────────┐ ┌────────┐ ┌─────────┐  │
│  │ RedoxFS │ │ドライバ │ │ネット   │  │
│  │(ファイル│ │デーモン │ │デーモン │  │
│  │システム)│ │        │ │        │  │
│  └────────┘ └────────┘ └─────────┘  │
├─────────────────────────────────────┤
│           カーネル空間               │
│  ┌────────────────────────────────┐  │
│  │    マイクロカーネル             │  │
│  │  メモリ管理・プロセス管理・IPC   │  │
│  └────────────────────────────────┘  │
└─────────────────────────────────────┘
```

### スキームベース IPC

Redox OS の最もユニークな設計が「スキーム（scheme）」。URL 的な表記でリソースやプロセスへのアクセスを統一する仕組み。

```
# 従来の Unix
open("/dev/tty")     # 特殊なデバイスファイル
open("/etc/hosts")   # 通常のファイル

# Redox OS のスキーム
open("display:")     # ディスプレイデーモンへのアクセス
open("tcp:example.com:80")  # TCP 接続
open("file:/etc/hosts")     # ファイルシステム
```

プログラムが `open("http://example.com")` を呼び出すと、カーネルはこれを `http` スキームデーモンに送るパケットに変換する。スキームデーモンは通常のユーザー空間プロセスであり、追加・変更が容易。

### RedoxFS

Redox OS の独自ファイルシステム。ZFS からインスパイアされた設計だが、マイクロカーネルアーキテクチャに適合させた実装。

- コピーオンライト
- スナップショット
- 整合性チェック

### 最新動向（2025〜2026年）

#### 2025年7月時点のパフォーマンス

ファイルコピーのパフォーマンスが 2024年11月比で **約 500〜700% 向上**。マルチスレッド I/O と非同期ドライバ改善による。

#### Servo Web エンジン稼働

2025年10月、Rust で書かれた Servo ウェブエンジンが Redox OS 上で動作を確認。Rust 製 OS 上で Rust 製ブラウザエンジンが動く節目。

#### 2025〜2026年ロードマップ

| 目標 | 内容 |
|------|------|
| セルフホスティング | Redox OS 上で Redox OS をコンパイル |
| ARM サポート | AArch64 の安定化 |
| COSMIC デスクトップ | System76 の COSMIC DE 統合 |
| Redox Server / Desktop | 用途特化バリアントの提供 |
| セキュリティ強化 | マイクロカーネルの利点を活かした分離強化 |

### Rust の活用

Redox OS のほぼすべてのコンポーネントが Rust で書かれており、メモリ安全性がシステムレベルで担保されている。カーネルのバッファオーバーフローや使用後メモリアクセスのような脆弱性が原理的に発生しない。

標準ライブラリ・ファイルシステムドライバ・ネットワークスタック・シェル・テキストエディタが含まれた完全なディストリビューションとして配布される。

### 現状と課題

Redox OS はまだ一般デスクトップ用途では成熟していない。ハードウェアサポートが限定的で、日常使いには Linux が現実的。研究・学習・新しい OS 設計の探求という文脈で価値が高い。

## ポイント

- Rust 製マイクロカーネル OS：カーネル空間をメモリ管理・プロセス管理・IPC のみに限定
- スキームベース IPC でドライバやデーモンをユーザー空間に隔離
- ファイルコピー 500〜700% 高速化など 2025 年も活発に開発継続
- Rust の安全性保証でシステムレベルの脆弱性を原理的に排除
- 日常使いより研究・教育目的向け（現時点）
