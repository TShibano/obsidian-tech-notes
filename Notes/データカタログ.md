## 概要

データカタログとは、組織が保有するデータ資産のメタデータを一元的に収集・管理・検索できるようにする仕組み（目録）である。データの所在・構造・所有者・品質・利用状況などのメタデータを体系的に整理することで、データの発見・理解・信頼性確保を支援し、データガバナンスの基盤となる。

## 詳細

### メタデータの種類

データカタログが管理するメタデータは主に3種類に分類される:

| 種類                | 説明              | 例                                |
| ----------------- | --------------- | -------------------------------- |
| **テクニカルメタデータ**    | データの物理的・技術的な属性  | テーブル名、カラム名、データ型、パーティション情報、ファイル形式 |
| **ビジネスメタデータ**     | データのビジネス上の意味・文脈 | データの定義、所有者、部署、用途、機密レベル           |
| **オペレーショナルメタデータ** | データの運用・利用に関する情報 | 更新履歴、アクセス頻度、クエリパターン、データリネージ      |

### 主要機能

#### データディスカバリ

自然言語検索やフィルタリングにより、組織内に散在するデータ資産を素早く発見できる。データエンジニアやデータサイエンティストがデータ探索に費やす時間を大幅に削減する。

#### データリネージ

データの生成元から最終的な利用先までの変換・移動の経路を可視化する。カラムレベルのリネージに対応するツールでは、個々の列がどのように変換されたかまで追跡できる。障害時の影響分析やデータ品質問題の原因特定に活用される。

#### データガバナンス

データのアクセス制御、プライバシー規制（GDPR 等）への準拠、データ品質の監視を統合的に管理する。データスチュワードの任命やポリシーの適用をカタログ上で一元管理できる。

#### データ品質モニタリング

データセットの品質指標（完全性、正確性、一貫性、鮮度）を継続的に監視し、異常を検知する。データプロファイリングにより統計情報を自動収集し、データの信頼性を可視化する。

### 主要ツール

#### オープンソース

| ツール                | 開発元           | 特徴                                                                                                                     |
| ------------------ | ------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **DataHub**        | LinkedIn      | ストリーミングファーストのアーキテクチャ。Kafka によるプッシュ・プル両対応のメタデータ取り込み。グラフ DB（Neo4j / JanusGraph）でエンティティ関係を管理                              |
| **OpenMetadata**   | Collate       | スキーマファースト・API ファーストの設計。90以上のコネクタ。MySQL / PostgreSQL + Elasticsearch のシンプルなスタック。[[Apache Airflow]] との統合によるプルベースのメタデータ収集 |
| **Apache Polaris** | Snowflake（寄贈） | [[Apache Iceberg]] 専用の REST カタログ。マルチエンジン相互運用性を提供。Spark、Flink、Trino 等から統一的にアクセス                                         |
| **Amundsen**       | Lyft          | Neo4j ベースのメタデータグラフ。シンプルな検索・ディスカバリに特化                                                                                   |

### DataHub vs OpenMetadata

2026年現在、オープンソースのデータカタログとして最も注目されている2つのツールの比較:

| 観点 | DataHub | OpenMetadata |
|------|---------|--------------|
| **アーキテクチャ** | 複雑。RDB + Elasticsearch + グラフ DB + Kafka | シンプル。RDB + Elasticsearch の4コンポーネント構成 |
| **メタデータ取り込み** | プッシュ・プル両対応（Kafka ストリーム） | プルベース（[[Apache Airflow]] による定期収集） |
| **リネージ** | エンドツーエンド + カラムレベル | エンドツーエンド + カラムレベル |
| **デプロイの容易さ** | コンポーネントが多くやや複雑 | コンポーネントが少なくシンプル |
| **拡張性** | 豊富な API（REST / GraphQL / Kafka） | スキーマファーストの拡張モデル |
| **コミュニティ** | 大規模。LinkedIn 発。GitHub Star 10k+ | 急成長中。GitHub Star 6k+ |

### 2026年のトレンド

#### AI 駆動のアクティブメタデータ

静的なカタログから、リアルタイムにメタデータを収集・分類・品質評価する「アクティブメタデータ」へ進化。AI/ML によるメタデータの自動抽出・分類・タグ付けにより、手動キュレーションの負担を大幅に軽減。

#### AI エージェントのデータ消費

2026年は AI エージェントが人間のアナリストと並んでデータの主要な消費者になりつつある。エージェントがデータを正しく理解・活用するために、セマンティックレイヤーやコンテキストエンジニアリングの重要性が高まっている。

#### 組み込み型 vs スタンドアロン型

メタデータを分析ワークフローの外部に置くスタンドアロン型では採用率が伸びにくいという課題から、変換・可視化・セキュリティ・AI の各レイヤーにメタデータを組み込む「エンベデッドカタログ」のアプローチが注目されている。

### ユースケース

- **データディスカバリの効率化**: 分析者がデータを探す時間を削減し、データ活用を促進
- **データガバナンスの実施**: アクセス制御、データ品質、プライバシー規制の遵守を一元管理
- **データパイプラインの影響分析**: リネージを活用して、スキーマ変更やデータ品質問題の影響範囲を特定
- **ML/AI パイプラインの管理**: モデルの学習データ、特徴量、実験結果のメタデータを追跡
- **データ民主化**: 非技術者でもデータの意味・品質・所在を理解できるようにする

## ポイント

- 組織のデータ資産のメタデータ（テクニカル・ビジネス・オペレーショナル）を一元管理する仕組み
- データディスカバリ、リネージ、ガバナンス、品質モニタリングが主要機能
- OSS では DataHub（ストリーミングファースト）と OpenMetadata（スキーマファースト）が二大勢力
- [[Apache Iceberg]] のカタログ層として Apache Polaris が標準化を進めている
- 2026年は AI 駆動のアクティブメタデータと、AI エージェントのデータ消費への対応がトレンド
