## 概要

DML（Data Manipulation Language）．既存のテーブルに対してデータの検索・追加・更新・削除を行うための SQL の分類．実行される SQL の約 90% が DML であり，最も頻繁に使用される SQL 分類．

## 詳細

### 主な DML コマンド

| コマンド | 説明 |
|---|---|
| `SELECT` | データの検索・取得 |
| `INSERT` | データの追加 |
| `UPDATE` | データの更新 |
| `DELETE` | データの削除 |

### 使用例

```sql
-- データ検索
SELECT id, name, email FROM users WHERE age > 20;

-- データ追加
INSERT INTO users (name, email, age) VALUES ('田中', 'tanaka@example.com', 30);

-- データ更新
UPDATE users SET age = 31 WHERE id = 1;

-- データ削除
DELETE FROM users WHERE id = 1;
```

### SQL 言語分類における位置付け

| 分類 | 略称 | 主なコマンド |
|---|---|---|
| [[データ定義言語]] | DDL | CREATE, ALTER, DROP, TRUNCATE |
| データ操作言語 | DML | SELECT, INSERT, UPDATE, DELETE |
| [[データ制御言語]] | DCL | GRANT, REVOKE |
| [[トランザクション制御言語]] | TCL | BEGIN, COMMIT, ROLLBACK |

### データエンジニアリングでの役割

- **データ取り込み**: `INSERT` や `UPSERT`（MERGE）でデータを格納
- **変換処理**: `SELECT` + `INSERT INTO ... SELECT` でデータを変換・集計
- **バッチ処理**: DML を定期実行してデータウェアハウスを更新
- **ELT**: Load 後に DML で変換を実施

### DMLとトランザクション

DML の変更はトランザクション内で実行され，[[トランザクション制御言語]] (TCL) の `COMMIT` で確定，`ROLLBACK` で取り消しが可能．

## ポイント

- SELECT は DML に含まれるが，データを変更しない（読み取り専用）
- `DELETE` と `TRUNCATE` の違い：DELETE は DML（ロールバック可），TRUNCATE は DDL（ロールバック不可）
- 大量データの INSERT は [[ETL]]/[[ELT]] パイプラインの主要処理
