## 概要

ブロックストレージは，データを固定長のブロック（通常 4KB〜1MB）単位で読み書きするストレージ方式．OS はブロックデバイスをファイルシステムや DB として利用する．HDD・SSD の物理的な読み書き単位であり，クラウドでは EBS（AWS）などが該当する．

## 詳細

### ストレージの種類と比較

| 種類 | アクセス単位 | プロトコル | 主な用途 |
|------|-----------|----------|---------|
| **ブロックストレージ** | ブロック（固定長） | iSCSI, FC, NVMe-oF | OS ディスク, DB, VM イメージ |
| **[[NAS]]**（ファイルストレージ） | ファイル | NFS, SMB/CIFS | 共有ファイルサーバ |
| **[[オブジェクトストレージ]]** | オブジェクト（可変長）+メタデータ | HTTP(S) REST API | 静的ファイル, バックアップ, 大規模データレイク |

### クラウドのブロックストレージ

| サービス | クラウド | 特徴 |
|---------|---------|------|
| Amazon EBS | AWS | SSD（gp3, io2）/ HDD（st1, sc1）の複数タイプ |
| Azure Managed Disks | Azure | Premium SSD, Standard HDD 等 |
| Persistent Disk | GCP | ゾーン / リージョナルの選択可 |

### 特性

- **低レイテンシ**: ブロック単位の直接アクセスでランダム I/O が高速
- **排他的マウント**: 通常 1 VM に 1 ボリュームとしてアタッチ（共有には特殊構成が必要）
- **ファイルシステムが必要**: OS でフォーマット（ext4, xfs, NTFS 等）してから利用

### HDFS との関係

[[HDFS]] は独自の分散ファイルシステムをブロック（128MB デフォルト）単位で実装している．クラウド移行では HDFS をオブジェクトストレージ（S3 等）に置き換えるケースが多い．

## ポイント

- DB・VM の OS ディスクにはブロックストレージが最適
- 複数サーバからの同時読み書きが必要な場合は NAS またはオブジェクトストレージを検討
- クラウドでは IOPS・スループットを追加費用で増強できる（プロビジョンド IOPS 等）
