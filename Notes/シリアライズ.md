## 概要

シリアライズ（Serialization）は，メモリ上のデータ構造やオブジェクトを，ファイル保存・ネットワーク転送・プロセス間通信に適したバイト列やテキスト形式に変換する処理．逆変換は [[デシリアライズ]]（Deserialization）という．

## 詳細

### 主なシリアライズ形式

| 形式 | 種別 | 特徴 | 主な用途 |
|------|------|------|---------|
| [[JSON]] | テキスト | 可読性高，言語横断，型情報なし | Web API, 設定ファイル |
| [[XML]] | テキスト | 厳密なスキーマ，冗長 | 企業システム，SOAP |
| [[CSV]] | テキスト | シンプル，表データ向き | データ分析，エクスポート |
| [[YAML]] | テキスト | 人間可読，階層構造 | 設定ファイル |
| [[TOML]] | テキスト | 設定ファイル特化，厳密な型 | 設定ファイル |
| Protocol Buffers | バイナリ | 高速，スキーマ必須 | [[RPC・gRPC]] |
| [[Apache Avro]] | バイナリ | スキーマ進化に強い | Kafka, Hadoop |
| [[Apache Parquet]] | バイナリ（列指向） | 分析クエリに最適 | データレイク, DWH |
| MessagePack | バイナリ | JSON 互換でコンパクト | ゲーム, IoT |

### テキスト vs バイナリの比較

| 観点 | テキスト（JSON 等） | バイナリ（Protobuf 等） |
|------|----------------|----------------|
| 可読性 | 高（デバッグしやすい） | 低（ツールが必要） |
| サイズ | 大きい | 小さい（数分の一） |
| 処理速度 | 遅い | 速い |
| スキーマ | 任意 | 必須（型安全） |

### シリアライズで考慮すべき点

- **スキーマ進化**: フィールド追加・削除時の後方・前方互換性
- **型精度**: JSON は 64bit 整数を正確に表せない（浮動小数点扱い）
- **セキュリティ**: デシリアライズは信頼できないデータへの適用で脆弱性になりやすい（Java Deserialization 等）

## ポイント

- API レスポンスは JSON，内部通信（マイクロサービス）は Protobuf，大規模データ処理は Parquet/Avro が定番
- [[圧縮]] と組み合わせることでさらにサイズを削減できる
