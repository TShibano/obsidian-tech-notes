## 概要

リレーショナルモデリング（Relational Modeling）は，Edgar F. Codd が1970年に集合論と述語論理に基づいて提唱したデータモデリング手法である．データをテーブル（リレーション）の集合として表現し，ER図（実体関連図）を用いて概念設計を行い，正規化によってデータの冗長性と不整合を排除する．

## 詳細

### 基本概念

| 概念 | 説明 | 例 |
|------|------|----|
| **リレーション（テーブル）** | 行と列で構成されるデータ集合 | 顧客テーブル，注文テーブル |
| **タプル（行）** | テーブルの1レコード | 特定の顧客1人のデータ |
| **属性（列）** | レコードの各項目 | 顧客ID，氏名，住所 |
| **主キー** | レコードを一意に識別する属性 | 顧客ID |
| **外部キー** | 他テーブルの主キーを参照する属性 | 注文テーブルの顧客ID |

### ER図（実体関連モデル）

[[ER図]]（Entity-Relationship Diagram）はリレーショナルモデリングの概念設計段階で使用する設計図．1975年にPeter Chenが考案．データベースの抽象的な構造を可視化したもので，データ(エンティティ)とデータ間関係(リレーションシップ)を表現する．

主要な要素:
- **エンティティ（Entity）**: 管理対象となるもの（顧客，注文，商品）
- **関連（Relationship）**: エンティティ間の関係（顧客が注文する，注文に商品が含まれる）
- **属性（Attribute）**: エンティティの特性（顧客の氏名・住所）
- **カーディナリティ**: 関係の多重度（1対1，1対多，多対多）

### 設計の3段階

```
要件定義
    ↓
概念モデル（ER図）── エンティティとその関係を抽象的に表現
    ↓
論理モデル（正規化）── テーブル構造・キー・制約を定義，DBMS非依存
    ↓
物理モデル── インデックス・パーティション等の実装詳細を定義
```

### 正規化

データの冗長性・更新異常を排除するプロセス．第1〜第3正規形（1NF〜3NF）が実践では主に使われる．4, 5はほぼ使われない．

| 正規形                 | 要件                            | 排除される問題 |
| ------------------- | ----------------------------- | ------- |
| **第1正規形（1NF）**      | 各セルに1つの原子値のみ，繰り返しグループなし       | 配列・多値属性 |
| **第2正規形（2NF）**      | 1NF + 非キー属性が主キー全体に関数従属        | 部分関数従属  |
| **第3正規形（3NF）**      | 2NF + 非キー属性間の推移的関数従属なし        | 推移的依存   |
| **ボイスコッド正規形（BCNF）** | すべての関数従属の決定子が候補キー             | 残存異常    |
| **第4正規形（4NF）**      | 多値従属性が存在しない．独立した多値の組み合わせ爆発を防ぐ | 多値従属性   |
| **第5正規形（5NF）**      | 分解しても自然結合で完全に復元できる            | 結合従属    |

### DWH設計との対比

| 観点 | リレーショナルモデリング（3NF） | マルチディメンショナルモデル（スタースキーマ） |
|------|------------------------------|------------------------------------------|
| 正規化 | 正規化（冗長排除） | 非正規化（クエリ速度重視） |
| 主な用途 | OLTP（業務処理） | OLAP（分析処理） |
| Inmon/Kimball | [[Inmonのトップダウン]] が採用 | [[Kimballのボトムアップ]] が採用 |
| 更新効率 | 高い | 低い |
| 読み取り速度 | 低い（結合多） | 高い（ジョイン少） |

### 変更のトラッキング
- 時間の経過とともにどのようにデータが変化したのかを履歴テーブルに保持する
- 履歴テーブルには，元のテーブルの構造をベースに，変更トラッキングするための列(変更日時，変更を行なったユーザID，各列の変更前後の値)を追加したテーブル
## ポイント

- Codd の関係モデルが全ての RDBMS の理論的基礎
- ER図で概念設計し，正規化で論理設計するのがリレーショナルモデリングの基本フロー
- 3NF までの正規化が実用的なバランス（それ以上は分析ワークロードには逆効果）
- DWH での分析用途には非正規化した [[マルチディメンショナルモデル]] が有効
- [[NoSQL]] の台頭でリレーショナルモデルの適用範囲が明確化（トランザクション重視な場面に特化）
