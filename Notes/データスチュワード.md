## 概要

データスチュワード（Data Steward）は，特定のデータドメイン（顧客データ，商品データ等）に対して責任を持ち，そのデータの品質・定義・ルール・メタデータを日常的に管理するデータガバナンスの実務担当者である．ビジネス部門側のロールとして，データの「意味」と「品質」を守る番人的な役割を担う．

## 詳細

### 役割と責務

| 責務 | 内容 |
|------|------|
| **データ定義の管理** | ビジネス用語・指標の定義を文書化・維持 |
| **データ品質監視** | 担当データの品質ルール定義・モニタリング・改善 |
| **メタデータ管理** | [[データカタログ]] へのメタデータ登録・更新 |
| **アクセス制御の運用** | データアクセス権限のレビュー・承認 |
| **問題解決** | データ品質問題の調査・報告・解決調整 |
| **ポリシー準拠** | データガバナンスポリシーへの準拠確認 |

### データオーナー・データアーキテクトとの違い

| ロール | 位置づけ | 主な責任 |
|--------|---------|---------|
| **データオーナー** | 経営・部門長レベル | データの最終責任者（意思決定権限あり） |
| **データスチュワード** | ビジネス現場レベル | 日常的な品質・定義管理（実務担当） |
| **[[データアーキテクト]]** | IT・技術レベル | データ構造・技術基盤の設計 |

### ドメインスチュワードシップ

大規模組織では，データドメインごとにスチュワードを配置する：

```
CDO
    ├── 顧客データスチュワード（マーケティング部門）
    ├── 製品データスチュワード（製品管理部門）
    ├── 財務データスチュワード（経理部門）
    ├── HR データスチュワード（人事部門）
    └── サプライチェーンデータスチュワード（物流部門）
```

### データメッシュとの関係

[[データメッシュ]] のアーキテクチャでは，各ドメインチームが「データプロダクトオーナー」として，データスチュワードに相当する責任を持つ設計になっている．

## ポイント

- データスチュワードは「IT 寄り」の [[データアーキテクト]] に対して「ビジネス寄り」のデータ品質番人
- [[データカタログ]] の実際のコンテンツを育てていく主体がデータスチュワード
- [[データガバナンス]] を形骸化させないために，現場に近いスチュワードの存在が不可欠
- 技術スキルより，ビジネス知識・コミュニケーション能力・データ品質への関心が重要
