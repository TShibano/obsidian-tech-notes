## 概要

プロジェクトマネージャー（PM: Project Manager）は，特定のプロジェクトの計画・実行・完了を管理し，スコープ・スケジュール・コスト・品質・リスクをコントロールする役割である．「いつまでに，どうやって作るか（When・How）」を担い，プロジェクトを目標通りに完遂させることに責任を持つ．

## 詳細

### プロダクトマネージャーとの違い

| 観点 | プロジェクトマネージャー（PM） | [[プロダクトマネージャー]]（PdM） |
|------|--------------------------|------------------------------|
| 責任対象 | プロジェクト（時限的な取り組み） | プロダクト（継続的な製品・サービス） |
| 主な問い | When・How（いつ・どう作るか） | Why・What（なぜ・何を作るか） |
| 期間 | プロジェクト開始〜完了 | 製品のライフサイクル全体 |
| 成功指標 | 納期・コスト・品質の達成 | ユーザー価値・ビジネス成果 |
| 視野 | 実行・管理・進捗 | 戦略・市場・ユーザーニーズ |

### 主な職務

| 責務 | 内容 |
|------|------|
| **計画立案** | WBS 作成，スケジュール策定，リソース計画 |
| **進捗管理** | マイルストーン確認，遅延の早期検知と対策 |
| **リスク管理** | リスクの識別・評価・対応策の策定 |
| **コスト管理** | 予算策定・実績追跡・EVM |
| **ステークホルダー管理** | 関係者との期待値調整，コミュニケーション |
| **チームマネジメント** | チームビルディング，課題解決，モチベーション維持 |

### プロジェクトマネジメントの主要フレームワーク

| フレームワーク | 特徴 |
|-------------|------|
| **PMBOK（PMI）** | プロジェクトマネジメントの知識体系．計画重視のウォーターフォール型 |
| **PRINCE2** | 欧州発の体系的プロセス志向のフレームワーク |
| **アジャイル/スクラム** | 反復開発で変化に適応．[[スクラム]] の スクラムマスターが PM 的役割を担うことも |
| **カンバン** | フロー管理に特化．WIP 制限でボトルネックを可視化 |

### アジャイルにおけるPMの役割変化

[[スクラム]] の普及により，伝統的な「指示型 PM」から「支援型リーダー（サーバントリーダー）」へのシフトが進んでいる．スクラムでは PM の役割は スクラムマスター・プロダクトオーナー（[[プロダクトマネージャー]]）に分割されることが多い．

## ポイント

- PM は「プロジェクトを予算・納期・品質通りに完遂する」ことが主要責任
- [[プロダクトマネージャー]] が「何を作るか」を決めるのに対し，PM は「どう作るか」を管理する
- PMBOK の知識体系が試験（PMP 資格）も含めた業界標準
- アジャイル時代は スクラムマスターとの役割融合・分担が進んでいる
