## 概要

リレーショナルデータベース（RDB: Relational Database）は，行と列からなるテーブル（関係）にデータを格納し，テーブル間の関係（リレーション）をキーで表現するデータベースモデルである．1970年代にE.F.Coddが提唱した関係モデルに基づき，SQL で操作する．

## 詳細

### 基本概念

| 概念 | 説明 |
|------|------|
| **テーブル（関係）** | 行と列からなる二次元の表形式データ構造 |
| **主キー（PK）** | 各行を一意に識別するカラムまたはカラムの組 |
| **外部キー（FK）** | 他テーブルの主キーを参照してリレーションを定義する |
| **インデックス** | 検索を高速化するためのデータ構造 |
| **正規化** | データの冗長性を排除し整合性を保つ設計手法 |

### ACID 特性

RDBMSは [[アトミックトランザクション]] に代表される ACID 特性（原子性・一貫性・独立性・永続性）を保証し，ビジネスクリティカルなデータ管理に適している．

### 主要製品

| 製品              | 種別  | 特徴                         |
| --------------- | --- | -------------------------- |
| [[PostgreSQL]]  | OSS | 標準SQL準拠，拡張性が高い             |
| MySQL           | OSS | Webアプリで広く使われる              |
| Oracle Database | 商用  | エンタープライズ向け，高機能             |
| SQL Server      | 商用  | Windows環境・Microsoft製品との親和性 |
| SQLite          | 組込み | サーバー不要，ファイルベース             |

### [[NoSQL]] との比較

| 観点 | RDB | NoSQL |
|------|-----|-------|
| データモデル | 表形式（固定スキーマ） | 多様（ドキュメント，KV，グラフ等） |
| スキーマ | スキーマオン・ライト | スキーマオン・リード（柔軟） |
| トランザクション | ACID | 多くはBASE（結果整合性） |
| スケーリング | 垂直スケーリングが主 | 水平スケーリングに適する |
| 適用 | 複雑なJOIN・集計，整合性重視 | 大規模・高スループット，柔軟スキーマ |

## ポイント

- SQL の標準化された構文は製品間で基本的に共通なため，ポータビリティが高い
- 正規化によりデータの冗長性を排除できるが，JOINコストが増えるトレードオフがある
- OLTPは RDB が主流，OLAPは列指向DWHへ移行する傾向がある
- 分散環境での水平スケーリングはRDBが苦手とする領域で，[[NewSQL]] がこれを解決しようとしている
