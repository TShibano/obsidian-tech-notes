## 概要

データアナリスト（Data Analyst）は，ビジネス課題の解決や意思決定支援を目的として，データを収集・整理・分析し，洞察を可視化・レポートする役割である．ビジネスサイドとデータの橋渡し役として，SQL・BIツール・統計知識を活用し，分析結果をビジネスユーザーが理解しやすい形で提供する．

## 詳細

### データサイエンティストとの違い

| 観点 | データアナリスト | [[データサイエンティスト]] |
|------|---------------|---------------------|
| 主な目的 | ビジネス課題の理解・可視化・レポート | 予測モデルの構築・統計的推論 |
| 分析手法 | 記述統計，集計，トレンド分析 | 機械学習，深層学習，高度な統計 |
| 主なツール | SQL，Excel，BIツール（Tableau，Power BI） | Python（pandas，scikit-learn），R |
| 機械学習 | 基本的な知識（使う程度） | 設計・実装レベル |
| ビジネス寄り度 | 高い | 中程度 |
| 数学・統計要求 | 基礎〜中程度 | 高い |

### 主な職務

| 業務 | 内容 |
|------|------|
| **データ収集・整備** | 必要なデータを特定し，SQL でデータを抽出・加工 |
| **記述分析** | KPI 集計，トレンド分析，異常値検出 |
| **可視化・レポート** | BI ツールでダッシュボード作成，経営報告資料作成 |
| **仮説検証** | A/Bテスト，相関分析，簡易統計検定 |
| **ビジネス提言** | 分析結果からのインサイト抽出と施策提案 |

### 必要なスキルセット

- **SQL**: データ抽出・集計クエリ（必須）
- **BI ツール**: Tableau，Power BI，Looker，Metabase
- **スプレッドシート**: Excel，Google Sheets（ピボットテーブル等）
- **統計基礎**: 平均・分散・相関・回帰の基礎
- **ドメイン知識**: 担当ビジネス領域の深い理解
- **コミュニケーション**: ビジネスユーザーへの説明・プレゼン

### データエンジニアとの連携

[[データエンジニア]] が整備した DWH・データマートを活用して分析を行う．アナリストが「このデータが欲しい」と要件を出し，エンジニアがパイプラインを構築するという協働が一般的．

## ポイント

- ビジネス課題の理解とデータ活用能力のバランスが重要
- SQL と BI ツールが基本ツールセット（機械学習より分析・可視化が中心）
- [[データサイエンティスト]] との違いは「ビジネス視点の記述分析」か「予測モデル構築」か
- 分析結果を「アクション可能なインサイト」として伝えるコミュニケーション能力が鍵
