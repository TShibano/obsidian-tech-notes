## 概要

IoT（Internet of Things，モノのインターネット）とは，家電・工場設備・自動車・医療機器などの物理的なモノがインターネットに接続し，センサーデータの収集・制御・連携を行う技術とその概念．

## 詳細

### システムアーキテクチャ

一般的な IoT システムは3層で構成される：

| 層 | 内容 |
|----|------|
| **デバイス層** | センサー，アクチュエーター，組み込みシステム |
| **ゲートウェイ・ネットワーク層** | [[IoTゲートウェイ]]，通信プロトコル変換，エッジ処理 |
| **クラウド・アプリケーション層** | データ収集・分析・可視化，制御コマンド送信 |

IoT World Forum の7層モデルではより細かく分類される（物理デバイス層，接続層，フォグコンピューティング層，データ蓄積層，データ抽象化層，アプリケーション層，コラボレーション層）．

### 主要プロトコル

| プロトコル | 特徴 | 用途 |
|-----------|------|------|
| **MQTT** | 軽量 pub/sub，低消費電力 | センサーデータ収集，IoT 標準 |
| **HTTP/HTTPS** | 汎用，広い対応 | REST API 連携 |
| **AMQP** | 信頼性の高いメッセージング | 業務システム連携 |
| **CoAP** | HTTP の UDP 版，超軽量 | 組み込みデバイス |
| **Bluetooth LE** | 近距離低消費電力 | ウェアラブル，スマートホーム |
| **LoRaWAN** | 広域低消費電力 | 農業・インフラ監視 |

### 活用分野

- **スマート製造**: 設備の稼働監視，予知保全（Predictive Maintenance）
- **スマートシティ**: 交通・電力・廃棄物管理の最適化
- **農業 IoT（AgriTech）**: 土壌・気象センサーによる精密農業
- **医療 IoT（IoMT）**: ウェアラブルによるバイタル監視
- **スマートホーム**: 家電制御，セキュリティ

### セキュリティ（2025年動向）

2025年3月，日本で経済産業省・IPA が「セキュリティ要件適合評価及びラベリング制度（JC-STAR）」を開始．IoT 製品のセキュリティ機能を ★1〜★4 の4段階で評価・ラベル表示する．

## ポイント

- エッジ処理（[[IoTゲートウェイ]]）でクラウド転送量を削減し，遅延とコストを抑制する
- MQTT が IoT 通信の事実上の標準プロトコルとなっている
- デバイスのリソース制約（CPU・メモリ・電力）が設計の最大制約条件
- セキュリティ：認証・暗号化・ファームウェア更新管理が重要
