## 概要

DDL（Data Definition Language）．データベースのデータ構造（スキーマ）を定義・変更・削除するための SQL の分類．テーブル・インデックス・ビューなどのオブジェクトを作成・管理するために使用する．

## 詳細

### 主な DDL コマンド

| コマンド | 説明 |
|---|---|
| `CREATE` | テーブル・ビュー・インデックス等を新規作成 |
| `ALTER` | 既存オブジェクトの構造を変更（カラム追加・削除等） |
| `DROP` | オブジェクトを削除 |
| `TRUNCATE` | テーブルの全データを高速削除（構造は保持） |

### 使用例

```sql
-- テーブル作成
CREATE TABLE users (
    id       INTEGER PRIMARY KEY,
    name     VARCHAR(100) NOT NULL,
    email    VARCHAR(255) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- カラム追加
ALTER TABLE users ADD COLUMN age INTEGER;

-- テーブル削除
DROP TABLE users;
```

### SQL 言語分類における位置付け

SQL は以下の4種類に分類される：

| 分類 | 略称 | 主なコマンド |
|---|---|---|
| データ定義言語 | DDL | CREATE, ALTER, DROP, TRUNCATE |
| [[データ操作言語]] | DML | SELECT, INSERT, UPDATE, DELETE |
| [[データ制御言語]] | DCL | GRANT, REVOKE |
| [[トランザクション制御言語]] | TCL | BEGIN, COMMIT, ROLLBACK |

### データエンジニアリングでの役割

- **スキーマ管理**: パイプラインのターゲットテーブルを DDL で定義
- **スキーマ進化**: `ALTER TABLE` でカラム追加などスキーマ変更を管理
- **マイグレーション**: dbt 等のツールで DDL 変更をバージョン管理

## ポイント

- DDL は即座にコミットされるため，`ROLLBACK` が効かない DBMS が多い（自動コミット）
- `DROP` と `TRUNCATE` の違い：DROP はオブジェクト自体を削除，TRUNCATE はデータのみ削除
- [[スキーマ]]定義の中核がDDL
