## 概要

CDC（Change Data Capture，変更データキャプチャ）は，データベースで発生した INSERT・UPDATE・DELETE の変更を検知し，その差分データのみをリアルタイムに取得・配信する技術である．バッチで全量比較する従来手法と比べて低負荷かつ低遅延でデータを同期できる．

## 詳細

### 実装方式

| 方式 | 仕組み | 特徴 |
|------|--------|------|
| **ログベース** | WAL（Write-Ahead Log）やbinlogを読み取る | 低負荷・高精度．最も広く使われる |
| **トリガーベース** | DBトリガーで変更をサイドテーブルに記録 | DB非依存だが書き込み負荷が増大 |
| **タイムスタンプベース** | 更新日時カラムで差分を検出 | 実装簡単だがDELETEを検出できない |

### 主要ツール

- **Debezium**: オープンソースのCDCフレームワーク．[[Apache Kafka]] Connect と組み合わせて広く使われる
- **AWS DMS**: マネージドCDCサービス
- **BigQuery CDC**: BigQueryへの直接CDC取り込み（GA済み）

### データの流れ

```
ソースDB（WAL/binlog）
  → CDC ツール（Debezium 等）
  → メッセージキュー / イベントストリーム（Kafka 等）
  → 宛先（DWH，検索インデックス，他サービス等）
```

## ポイント

- ログベース CDC はソース DB への負荷がほぼゼロで，ミリ秒単位の低遅延レプリケーションを実現する
- DB 移行・クラウド移行時のダウンタイム最小化（ライブマイグレーション）に有効
- マイクロサービス間のデータ同期，検索インデックス更新，DWH へのリアルタイムロードなど幅広い用途に使われる
- イベントのスキーマ変化への対応（スキーマエボリューション）が運用上の課題になりやすい
