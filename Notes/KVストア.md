## 概要

KVストア（Key-Value Store / キーバリューストア）は，データをキーと値（バリュー）のペアで格納するシンプルな構造の NoSQL データベースである．最もシンプルなデータモデルを持ち，キーによる超高速な読み書きが特徴で，キャッシュ・セッション管理・リアルタイムランキングなど，低レイテンシが求められる用途に広く採用される．

## 詳細

### データモデル

```
キー          → バリュー
"user:1001"  → { "name": "田中太郎", "score": 1500 }
"session:abc" → "eyJhbGciOiJIUzI1NiJ9..."
"rank:2026-05-05" → [["Alice", 9800], ["Bob", 8500], ...]
```

バリューは文字列・JSON・バイナリ・リスト・セット・ソート済みセットなど，実装によって多様な型をサポートする．

### RDBMS との比較

| 観点 | KVストア | RDBMS |
|------|---------|-------|
| クエリ方法 | キー指定のみ（範囲スキャンは実装依存） | 任意の条件で SQL クエリ |
| スキーマ | なし | 固定 |
| 速度 | 超高速（インメモリは特に） | 中程度 |
| 一貫性 | 結果整合性が多い | ACID 保証 |
| 用途 | キャッシュ，セッション，低レイテンシ | 複雑な業務ロジック，トランザクション |

### 主要な KVストア

| 製品 | 特徴 | 主な用途 |
|------|------|---------|
| **Redis** | インメモリ．豊富なデータ型（String/List/Set/ZSet/Hash）．パブサブ | キャッシュ，セッション，リアルタイムランキング |
| **Amazon DynamoDB** | フルマネージド．KV＋ドキュメント．スケール無制限 | 大規模 Web アプリ，IoT，サーバーレス |
| **Memcached** | シンプルなインメモリキャッシュ．分散対応 | 分散キャッシュ |
| **etcd** | 分散 KVストア．Raft による強一貫性 | Kubernetes クラスタの設定・サービスディスカバリ |
| **Apache Cassandra** | 大規模分散 KV + ワイドカラム | 大量書き込み，高可用性 |

### Redis の主なデータ型

| 型 | 説明 | ユースケース |
|----|------|-----------|
| **String** | 文字列（JSON 含む） | キャッシュ，カウンタ |
| **List** | 順序付きリスト | キュー，タイムライン |
| **Set** | 重複なし集合 | タグ管理，ユニークユーザー数 |
| **Sorted Set（ZSet）** | スコア付きセット | リアルタイムランキング |
| **Hash** | フィールド→バリューの辞書 | ユーザーオブジェクト |
| **Stream** | 時系列ログ | イベントログ，メッセージキュー |

### ユースケース

- **セッション管理**: ログインセッション ID → ユーザー情報のマッピング（TTL 付き）
- **キャッシュ**: DB クエリ結果のキャッシュ（レイテンシ削減）
- **リアルタイムランキング**: Redis の Sorted Set でスコア集計
- **レート制限**: 分間リクエスト数のカウント（Redis の INCR + TTL）
- **パブサブ**: サービス間のリアルタイムメッセージング

## ポイント

- 「キーでバリューを取得する」というシンプルな操作に特化することで超高速を実現
- Redis はインメモリのため，再起動時のデータ揮発に注意（RDB/AOF による永続化で対応）
- [[ポリグロット永続化]] では KVストアは RDBMS やドキュメントDB と組み合わせてキャッシュ層として使うのが典型パターン
- DynamoDB は KVストアでありながら大規模スケールと耐久性を持つ「本番 DB としての KVストア」
