## 概要

データガバナンス（Data Governance）は，組織のデータ資産の可用性・整合性・セキュリティ・使いやすさを確保するための，ポリシー・プロセス・役割・標準の体系である．DAMA-DMBOK において「データマネジメント全体を統括する中心的な知識領域」と位置づけられており，データの品質・セキュリティ・コンプライアンス・倫理的利用を組織全体で一貫して管理する．

## 詳細

### DMBOK におけるデータガバナンスの位置づけ

[[DMBOK]] では11の知識領域を「DAMAホイール」で表現し，データガバナンスを中心（ハブ）に配置する：

```
        データアーキテクチャ
        データモデリング
        データストレージ
        データセキュリティ
        データ統合・相互運用性
── [データガバナンス（中心）] ──
        データ品質
        データウェアハウジング・BI
        メタデータ管理
        参照・マスターデータ管理
        文書・コンテンツ管理
```

### データガバナンスの主要コンポーネント

| コンポーネント          | 内容                              |
| ---------------- | ------------------------------- |
| **ポリシー・標準**      | データの収集・利用・保管・廃棄に関するルール          |
| **データスチュワードシップ** | 各データドメインの責任者（[[データスチュワード]]）の設置  |
| **データカタログ**      | [[データカタログ]] によるデータ資産の発見・理解のサポート |
| **データ品質管理**      | [[データ品質]] 基準の定義・計測・改善           |
| **データリネージ**      | [[データリネージ]] でデータの出所と変換履歴を追跡     |
| **アクセス制御**       | 誰がどのデータにアクセスできるかの権限管理           |
| **コンプライアンス**     | GDPR，個人情報保護法等の法規制への対応           |

### 主要なフレームワーク

| フレームワーク | 提供元 | 特徴 |
|-------------|-------|------|
| **DAMA-DMBOK** | DAMA International | データガバナンスを含む11知識領域のベストプラクティス集 |
| **DCAM** | EDM Council | 金融機関向け．データ品質・リスク管理に強い |
| **Salesforce Data Governance** | Salesforce | Salesforce エコシステム向けフレームワーク |
| **TOGAF** | The Open Group | EA全体の中でデータガバナンスを位置づける |

### ガバナンスの組織体制

```
CDO（最高データ責任者）
    ├── データガバナンス委員会（Executive）
    │       ポリシー承認・優先順位決定
    ├── データオーナー（各ビジネス部門）
    │       データの最終責任者
    ├── データスチュワード（各ドメイン）
    │       日常的なデータ品質・定義管理
    └── データエンジニア（IT）
            技術的な実装・パイプライン管理
```

### データメッシュとのガバナンス

[[データメッシュ]] では「連邦型計算ガバナンス」を採用し，全社共通ポリシーとドメイン自律性を両立させる．自動化されたポリシー適用（コンプライアンスのコード化）がトレンド．

## ポイント

- データガバナンスは「データ品質を誰が何に基づいて管理するか」を定める体制設計
- 技術ツール（カタログ，リネージ等）より，ポリシー・役割・プロセスの整備が先決
- CDO（最高データ責任者）の設置と経営レベルのコミットメントが成功の鍵
- GDPR・個人情報保護法などの規制対応においてもデータガバナンスは基盤となる
- 2026年のトレンド：AI によるガバナンスの自動化，データメッシュ下での連邦ガバナンス
