## 概要

グラフデータベースは，データを**ノード（頂点）**と**エッジ（辺）**と**プロパティ**で表現するデータベースである．エンティティ間の「関係性」を第一級の概念として扱い，深い多段結合を高速に処理できる．ソーシャルグラフ，知識グラフ，不正検知などで威力を発揮する．

## 詳細

### データモデル

```
(ユーザー:Alice {age: 30}) -[FOLLOWS]-> (ユーザー:Bob {age: 25})
(Bob) -[LIKES]-> (投稿:P1 {content: "..."})
```

- **ノード**: エンティティ（人，物，場所など）
- **エッジ**: ノード間の有向・無向の関係（「フォローする」「購入した」など）
- **プロパティ**: ノード・エッジに付随する属性（名前，日時，重みなど）
- **ラベル**: ノードやエッジの種類を分類するタグ

### RDBとの比較

| 観点 | RDB（JOIN） | グラフDB |
|------|-------------|---------|
| 多段結合 | テーブル増加でO(n^m)に劣化 | ノード追跡でほぼO(n) |
| スキーマ | 固定スキーマ | 柔軟（ノード種類を追加しやすい） |
| 関係性の表現 | 外部キー・中間テーブル | エッジとして直接表現 |
| 適用領域 | 集計・分析 | 関係探索・推薦・経路探索 |

### 代表的な製品

| 製品 | 特徴 |
|------|------|
| **Neo4j** | 最も普及したグラフDB．クエリ言語 Cypher を使う |
| **Amazon Neptune** | AWSのマネージドグラフDB．SPARQL・Gremlin対応 |
| **ArangoDB** | グラフ・ドキュメント・KV のマルチモデルDB |
| **TigerGraph** | エンタープライズ向け高性能グラフDB |

### クエリ言語（Cypher 例）

```cypher
// Aliceのフォロワーのフォロワーを探す（2ホップ）
MATCH (alice:User {name: "Alice"})<-[:FOLLOWS*2]-(follower)
RETURN follower.name
```

### ナレッジグラフとLLM

グラフDBは**ナレッジグラフ**の格納基盤として注目されており，[[RAG]] と組み合わせることで LLM のハルシネーション抑制に活用される（GraphRAG パターン）．

## ポイント

- 6次の隔たりのような深い多段探索は，RDBのJOINより圧倒的に高速
- ソーシャルネットワーク・コンプライアンス・不正検知・推薦システムで特に有効
- 全件集計（SUM，GROUP BY等）は RDB や列指向DBより苦手
- LLMとナレッジグラフを組み合わせる GraphRAG の文脈で再注目されている
