## 概要

SSH File Transfer Protocol．SSH の仕組みを利用してファイルを暗号化転送するプロトコル．FTP の安全な代替として広く使われ，データエンジニアリングにおける外部データソースからのデータ取り込みで頻繁に用いられる．

## 詳細

### FTP との違い

| | FTP | SFTP |
|---|---|---|
| 暗号化 | なし | SSH による全文暗号化 |
| ポート | 21 | 22（SSH と同じ） |
| 認証 | パスワード（平文） | パスワード＋公開鍵認証 |
| 安全性 | 低い | 高い |

### 仕組み

1. クライアントが SSH 接続（ポート 22）でサーバと接続を確立
2. SSH トンネル経由でファイル転送セッションを開始
3. すべてのデータ（ファイル内容＋コマンド）が暗号化された状態で転送

SSH クライアント・サーバアーキテクチャモデルを採用しており，接続・リソース配分・セキュリティ制御を提供する．

### 主な機能

- ファイルのアップロード・ダウンロード
- ディレクトリのナビゲーション・操作
- ファイルパーミッションの変更
- ファイル名変更・削除

### 認証方式

- **パスワード認証**: ユーザ名＋パスワード
- **公開鍵認証**: SSH 公開鍵・秘密鍵ペアによる認証（より安全）

### データエンジニアリングでの用途

- 外部ベンダーやパートナーからのデータ受け取り
- レガシーシステムとのデータ連携
- [[FTP]] の代替として既存フローの移行先

## ポイント

- 通信・ファイル内容すべてが暗号化されるためセキュリティが高い
- SSH の公開鍵認証と組み合わせるとパスワードレスで安全な自動化が可能
- データ転送プロトコルとしては [[SCP]] より機能が豊富
