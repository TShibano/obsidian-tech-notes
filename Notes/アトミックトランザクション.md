## 概要

アトミックトランザクション（Atomic Transaction）とは，複数のデータ操作を一つの不可分な単位として扱い，「全て成功する」か「全て失敗してロールバックする」かのいずれかしかない処理のことである．ACID 特性の「A（Atomicity，原子性）」に対応する．

## 詳細

### ACID 特性

| 特性 | 英語 | 意味 |
|------|------|------|
| **原子性** | Atomicity | トランザクション内の操作は全て実行されるか全て取り消される |
| **一貫性** | Consistency | トランザクション前後でデータの整合性が保たれる |
| **独立性** | Isolation | 並行トランザクションは互いに干渉しない |
| **永続性** | Durability | コミットされたデータは障害後も失われない |

### 実装の仕組み

**ログ先行書き込み（WAL: Write-Ahead Logging）**
- 変更内容をデータファイルに書き込む前にログに記録する
- undo ログでロールバック，redo ログでリカバリを実現する

**2フェーズコミット（2PC）**
- 分散トランザクションで複数ノード間の原子性を保証するプロトコル
- Prepare フェーズ → Commit/Abort フェーズの2段階で実行する

### 使用例：銀行送金

```sql
BEGIN;
  UPDATE accounts SET balance = balance - 1000 WHERE id = 'A';  -- 出金
  UPDATE accounts SET balance = balance + 1000 WHERE id = 'B';  -- 入金
COMMIT;  -- 両方成功で確定，途中エラーは ROLLBACK
```

どちらか一方だけ実行された場合，トランザクション全体がロールバックされる．

### 分離レベル

トランザクションの独立性のレベルは設定可能で，強い順に:

1. **SERIALIZABLE** — 完全な直列実行と同等
2. **REPEATABLE READ** — 同一クエリは同じ結果を返す
3. **READ COMMITTED** — コミット済みデータのみ読む
4. **READ UNCOMMITTED** — コミット前データも読む（ダーティリード発生）

## ポイント

- 原子性を保証することで「半端な状態」のデータがDBに残らなくなる
- ロールバックはアプリ側のエラーハンドリングを簡潔にする
- 分散システムでの原子性保証（2PC，Saga パターン等）は難易度が高い
- [[NoSQL]] の多くは ACID よりも BASE（結果整合性）を採用してスケーラビリティを優先する
