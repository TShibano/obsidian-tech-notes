## 概要

同一データリソースを複数のアプリケーション・ユーザ・組織が利用できるようにするプロセスおよび仕組み．データをコピーせずにセキュアにアクセスを提供することで，組織内外のデータ非集中化を実現する．

## 詳細

### 基本概念

- データエンジニアリングにおけるソースシステムとストレージの両フェーズで重要
- **コピーなし共有**: データを物理的にコピーせず，元データへのアクセス権を付与するのが現代的アプローチ
- データ共有により，[[データメッシュ]]や[[データマーケットプレイス]]のアーキテクチャが実現可能

### 主な実装方式

| 方式 | 概要 |
|---|---|
| Snowflake Secure Data Sharing | 同一リージョン内でコピーなし即時共有 |
| Delta Sharing | オープンプロトコル．プラットフォーム横断でデータ共有 |
| データ交換所 (Data Exchange) | 招待ベースでの選択的グループ共有 |
| AWS Data Exchange | マーケットプレイス型のデータ配信 |

### Delta Sharing プロトコル

- オープンな業界標準プロトコル（Databricks が発案）
- 計算プラットフォームに依存しない（Spark，pandas，Power BI など）
- REST API ベースで組織間のセキュアなデータ共有を実現

### データ提供の観点

- セキュリティ：データの機密性と完全性の確保
- ガバナンス：アクセス制御，監査ログ，データ品質の維持
- [[データ契約]]との連携：スキーマや品質の合意

## ポイント

- コピーなし共有はリアルタイム性と一貫性の両立に優れる
- 組織内のデータ非集中化を促進し，ドメイン別の自律的管理を支援
- データ提供者と消費者の間で[[データ契約]]を結ぶことが望ましい
