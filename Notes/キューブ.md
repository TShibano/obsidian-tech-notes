## 概要

OLAPキューブ（OLAP Cube）は，OLAP（オンライン分析処理）において使用される多次元データ構造である．データウェアハウスから集計したデータを複数のディメンション（軸）で事前に集計・整理し，高速な多次元分析を実現する．

## 詳細

### 概念

キューブという名前は「3次元の立方体」に由来するが，実際には3次元に限らず多数のディメンションを持てる（ハイパーキューブとも呼ばれる）．

```
例）売上キューブ（時間 × 地域 × 製品）

          製品A 製品B 製品C
東京 Q1  [100] [200] [150]
東京 Q2  [120] [180] [160]
大阪 Q1  [ 90] [210] [140]
大阪 Q2  [110] [220] [130]
```

各セル（メジャー）には売上金額・数量・利益などのビジネス指標が格納される．

### キューブの構成要素

| 要素 | 説明 | 例 |
|------|------|----|
| **メジャー（Measure）** | 分析対象の数値指標 | 売上金額，数量，利益 |
| **ディメンション（Dimension）** | 分析の切り口・軸 | 時間，地域，製品，顧客 |
| **階層（Hierarchy）** | ディメンション内の集計レベル | 年→四半期→月→日 |
| **メンバー（Member）** | 各ディメンションの値 | 2026年，東京，製品A |
| **セル（Cell）** | メジャーの格納単位 | 東京×Q1×製品A の売上 |

### OLAPの実装タイプ

キューブの格納・処理方式によって3種類に分類される．

| タイプ | 説明 | メリット | デメリット |
|--------|------|---------|-----------| 
| **MOLAP**（多次元 OLAP） | 専用の多次元DBに事前集計して格納 | 高速なクエリ応答 | ストレージ消費大，データ更新に時間 |
| **ROLAP**（リレーショナル OLAP） | RDBMS上で動的にキューブをシミュレート | 大容量データ対応 | クエリのたびに計算するため低速 |
| **HOLAP**（ハイブリッド OLAP） | MOLAP と ROLAP を組み合わせ | 速度とスケールを両立 | 構成が複雑 |

### キューブの主要操作

| 操作 | 説明 |
|------|------|
| **スライス（Slice）** | 1つのディメンションで値を固定して断面を取得（例：2026年のみ） |
| **ダイス（Dice）** | 複数ディメンションで部分集合を抽出（例：東京×Q1） |
| **ドリルダウン（Drill Down）** | 年→四半期→月 へと詳細に掘り下げ |
| **ロールアップ（Roll Up）** | 月→四半期→年 へと集約レベルを上げる |
| **ピボット（Pivot）** | 行・列の軸を入れ替えて別視点で分析 |

### データソースとの関係

キューブのデータは[[リレーショナルデータウェアハウス]]のスタースキーマ・スノーフレークスキーマから構築される:

```
[スタースキーマ（DWH）]
  ファクトテーブル（売上） + ディメンションテーブル（時間・地域・製品）
          ↓ ETL・集計
[OLAPキューブ]
  事前集計済みの多次元データ構造
          ↓
[BI ツール（Tableau，Power BI など）]
```

### 代表的な OLAP キューブ実装

- **Microsoft SQL Server Analysis Services（SSAS）**: MOLAP/ROLAP/HOLAP をサポートする Microsoft の OLAP エンジン
- **Oracle Essbase**: 大規模エンタープライズ向け MOLAP エンジン
- **Apache Kylin**: Apache Hadoop/Spark ベースの分散 OLAP エンジン
- **Druid**: リアルタイム OLAP 向けの列指向データストア
- **クラウド DWH（BigQuery，Snowflake など）**: ROLAP として機能し，キューブを内部で動的に実現

### 現代における位置づけ

クラウド DWH の普及により，専用キューブ（MOLAP）への事前集計よりも，Snowflake・BigQuery などのクラウド DWH が ROLAP として機能するアーキテクチャが主流になりつつある．BI ツール側でのインメモリ集計（Power BI の Tabular Model など）も一般的になっている．

## ポイント

- OLAP キューブは複数ディメンションにわたる事前集計済みデータ構造で，BI クエリを高速化する
- MOLAP（多次元DB）・ROLAP（RDBMS上）・HOLAP（ハイブリッド）の3タイプがある
- [[マルチディメンショナルモデル]]（スタースキーマ等）がキューブの論理的なデータ設計に対応する
- クラウド DWH の進化により，物理的なキューブ構築（MOLAP）の必要性は低下している
- [[OLAP]] の5大操作（スライス・ダイス・ドリルダウン・ロールアップ・ピボット）はキューブを前提に設計されている
