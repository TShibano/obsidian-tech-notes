## 概要

対称型マルチプロセシング（SMP: Symmetric Multiprocessing）は，複数のプロセッサが単一の共有メモリ空間を介して対等に協調動作する並列コンピューティング方式である．すべてのプロセッサが同一のメモリとI/Oデバイスへ均等にアクセスできる．

## 詳細

### アーキテクチャ

```
CPU-1 ─┐
CPU-2 ─┼─ [共有バス / インターコネクト] ─ 共有メモリ
CPU-3 ─┤                                  共有 I/O
CPU-4 ─┘
```

- 全 CPU が同一のメモリアドレス空間を共有（UMA: Uniform Memory Access）
- OSが複数 CPU へタスクを動的に割り当て
- プロセッサ間の主従関係がなく対等（対称）な関係

### 動作の仕組み

1. OS のスケジューラが処理負荷を複数 CPU に均等分散
2. 各 CPU は割り当てられたタスクを実行
3. 結果は共有メモリに書き込まれ，他 CPU からも参照可能
4. キャッシュコヒーレンスプロトコルで各 CPU のキャッシュ整合性を維持

### MPP との比較

| 項目 | SMP | MPP（超並列プロセシング） |
|------|-----|--------------------------|
| メモリ共有 | 共有（単一アドレス空間） | 非共有（各ノード独立） |
| スケーラビリティ | 垂直スケール（CPU数上限あり） | 水平スケール（ノード追加） |
| 通信 | 共有メモリ（高速） | ネットワーク（相対的に低速） |
| 複雑さ | 低い | 高い（データ分散設計が必要） |
| コスト | 高コアサーバー | 汎用サーバー多数 |
| 適用規模 | 中規模データ | 超大規模データ |

### データウェアハウスでの位置づけ

従来の [[リレーショナルデータウェアハウス]] は SMP アーキテクチャで実装されることが多かった（Oracle，SQL Server，PostgreSQL など）．しかし，データ量の増大に伴い [[超並列プロセシング]]（MPP）アーキテクチャへの移行が進んでいる．

### 用途

- **エンタープライズサーバー**: Webホスティング，アプリケーションサーバー
- **データベースサーバー**: 中規模データベースの高速処理
- **仮想化基盤**: VM ホストとして複数の仮想マシンを並列実行
- **科学技術計算**: 数値シミュレーション

## ポイント

- 共有メモリのボトルネックにより，CPU 数の増加に上限がある（一般的に 8〜32 コアが実用的上限）
- 単一障害点（共有バス・メモリ）のリスクがある
- プログラミングモデルが単純で，マルチスレッドプログラムがそのまま動作する
- クラウド時代は 1 台の大型 SMP より MPP クラスタが主流になりつつある
