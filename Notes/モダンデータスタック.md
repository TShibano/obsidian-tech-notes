## 概要

モダンデータスタック（Modern Data Stack，MDS）とは，データの収集・保存・変換・分析をクラウドネイティブなサービスの組み合わせで構築するデータ基盤の設計思想．単一ベンダーの垂直統合ではなく，各レイヤーのベストオブブリードツールをモジュール式に組み合わせる．

## 詳細

### レイヤー構成

| レイヤー | 役割 | 代表ツール |
|---------|------|-----------|
| **データ取り込み（Ingestion）** | ソースからDWHへのデータ転送 | Fivetran, Airbyte, Stitch |
| **ストリーミング取り込み** | リアルタイムデータ転送 | Apache Kafka, AWS Kinesis |
| **ストレージ（DWH/Data Lake）** | データの集中管理・保存 | Snowflake, BigQuery, Databricks, Redshift |
| **変換（Transform）** | DWH 内でのデータ加工・モデリング | [[dbt]] |
| **オーケストレーション** | パイプラインのスケジュール・監視 | Apache Airflow, Dagster, Prefect |
| **BI・可視化** | ダッシュボード・レポート | Looker, Tableau, Metabase, Redash |
| **データカタログ** | メタデータ管理・データ検索 | DataHub, Amundsen, Apache Atlas |

### 従来スタックとの比較

| 観点 | 従来 | モダンデータスタック |
|------|------|------------------|
| 構築 | オンプレミス中心 | クラウドネイティブ |
| 変換 | ETL（ロード前に変換） | [[ELT]]（ロード後に変換） |
| スケーリング | 事前に容量設計 | オンデマンド・弾力的 |
| ツール選択 | 単一ベンダー | ベストオブブリード |

### 2025年以降の動向

- AI/ML 統合：LLM を活用したデータ変換・メタデータ生成の自動化
- リアルタイム化：バッチ処理からストリーミング処理への移行加速
- データ民主化：セルフサービス BI の普及でエンジニア以外もデータ活用

## ポイント

- [[dbt]] がデータ変換の事実上の標準となっており，SQL で変換ロジックを記述できる
- ストレージと計算リソースの分離（Snowflake・BigQuery）がコスト最適化の鍵
- 各ツールが標準インターフェースで接続されているため，ツール入れ替えが容易
- [[DataOps]] の実践基盤としてパイプラインのバージョン管理・テストが重要
