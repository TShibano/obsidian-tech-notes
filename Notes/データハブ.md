## 概要

データハブ（Data Hub）は，データの作成者（プロデューサー）と利用者（コンシューマー）を仲介し，データ共有を可能にする論理的なアーキテクチャである．ハブ＆スポーク構造により，複数システム間のデータ統合・配信を一元管理する．
オペレーショナル用途向けにシステム間のデータのやり取りを行いたい場合に使われることが多く，データ分析等はモダンデータウェアハウスやデータレイクハウスなどが優れる．
ユーザによって単語の意味が変わる点に注意する，
## 詳細

### アーキテクチャ

データハブは「ハブ＆スポーク」モデルを採用する．各システム（スポーク）はハブを通じてデータを提供・取得する．

```
システムA ─┐
システムB ─┼─ [データハブ] ─┬─ BIツール
システムC ─┘               ├─ アプリケーション
                           └─ データウェアハウス
```

### 主な特性

| 特性 | 説明 |
|------|------|
| リアルタイム対応 | ビジネスのリアルタイムビューを最新状態に維持 |
| 双方向 | 上流システムへのデータ書き戻しも可能 |
| マルチモデル | 構造化・非構造化データを含む多様なデータ形式を格納 |
| ELT | 生データを取り込み，用途に応じてキュレーション |

### データウェアハウスとの違い

| 項目 | データハブ | データウェアハウス |
|------|-----------|------------------|
| 更新頻度 | リアルタイム | バッチ（日次等） |
| データ型 | 構造化・非構造化 | 主に構造化 |
| 利用者 | 業務ユーザー〜専門家 | データ分析専門家 |
| 主な用途 | 統合・配信・オペレーション | 履歴分析・BI |
| 書き戻し | 対応 | 非対応 |

### データレイクとの違い

データレイクが「生データの大規模保存」に特化するのに対し，データハブは「データの流通・仲介」に重点を置く．データハブはキュレーション・ガバナンス機能を備え，データレイクよりも整理されたデータを提供する．

### 用途

- **システム間データ連携**: ポイント・ツー・ポイント連携をハブに集約してスパゲッティを解消
- **マスターデータ管理**: 顧客・商品マスタのゴールデンレコードを一元管理
- **段階的システム移行**: レガシーシステムのデータをハブ経由で新システムへ移行
- **データ民主化**: セルフサービス型のデータアクセスを実現

## ポイント

- ハブ＆スポーク構造により，n×m の連携コストを n+m に削減できる
- [[データメッシュ]] との違いは中央集権（ハブ）vs. 分散（ドメイン）の思想
- [[マスターデータ管理]] の実装基盤として活用されるケースが多い
- [[ELT]] パターンと親和性が高い（生データ取り込み→後からキュレーション）
- データマーケットプレイスの基盤として機能することもある
