## 概要

IoTゲートウェイとは，IoTデバイス（センサー・設備）とクラウド・社内システムの間に位置し，データの収集・変換・フィルタリング・セキュリティ処理を担う中継機器．エッジコンピューティングの実行基盤としても機能する．

## 詳細

### 役割と機能

| 機能 | 内容 |
|------|------|
| **プロトコル変換** | デバイス側の Bluetooth LE・Zigbee・Modbus を MQTT・HTTP に変換 |
| **データ集約** | 複数デバイスのデータを集約してクラウド転送量を削減 |
| **エッジ処理** | 現場でのリアルタイム処理・異常検知・データ前処理 |
| **セキュリティ** | 認証・暗号化・ファイアウォール機能でデバイスを保護 |
| **バッファリング** | ネットワーク断絶時のデータ一時保存 |

### エッジコンピューティングとの関係

IoTゲートウェイはエッジコンピューティングの主要な実行基盤．すべてのデータをクラウドに送信すると通信量増大・遅延・コスト増が生じるため，ゲートウェイが現場近くで一次処理を行い，必要なデータのみをクラウドへ転送する．

```
[センサー群] → [IoTゲートウェイ] → [クラウド/データセンター]
                  ↑
         プロトコル変換・エッジ処理
         異常検知・データ圧縮
         バッファリング・セキュリティ
```

### IoTゲートウェイ vs IoTルーター

| 観点 | IoTゲートウェイ | IoTルーター |
|------|----------------|------------|
| データ処理 | エッジ処理・プロトコル変換あり | 基本的にルーティングのみ |
| 演算能力 | 高（Linux等を搭載） | 低〜中 |
| 用途 | 工場・農業・医療等の高度用途 | 単純な接続性確保 |

### 代表的な製品・ソフトウェア

- **ハードウェア**: Raspberry Pi（DIY），産業用ゲートウェイ（研華・コンテック等）
- **クラウドゲートウェイ**: AWS IoT Greengrass，Azure IoT Edge，Google Cloud IoT

## ポイント

- マルチプロトコル対応が選定の重要ポイント（現場デバイスの通信方式に依存）
- エッジ処理能力（CPU・メモリ・ストレージ）が処理可能な分析の複雑さを決める
- 屋外・工場環境では防塵・防水・温度耐性などの物理耐久性も考慮する
- [[IoT]] システム全体のセキュリティ境界として機能するため，ゲートウェイのセキュリティ強化が重要
