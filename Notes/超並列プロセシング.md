## 概要

超並列プロセシング（MPP: Massively Parallel Processing）は，多数の独立したノード（サーバー）がそれぞれ独自のメモリとストレージを持ち，ネットワークで協調してデータを並列処理するアーキテクチャである．大規模データウェアハウスやクラウドデータ基盤の中核技術として広く採用されている．

## 詳細

### アーキテクチャ

```
クライアント → [リーダーノード（クエリ分解・集約）]
                       ↓ 並列分散クエリ
         ┌─────────────┬─────────────┬─────────────┐
    [コンピュートノード1] [ノード2]    [ノード3]    [ノードN]
    (データ+CPU+メモリ)
         └─────────────┴─────────────┴─────────────┘
                    ↑ 各ノードが担当データを処理
```

MPP は「シェア・ナッシング（Share Nothing）」アーキテクチャを採用し，各ノードはメモリ・ストレージ・CPU を完全に独立して保有する．ノード間の通信はネットワーク経由のみ．

### SMP との比較

| 項目 | MPP | SMP（対称型マルチプロセシング） |
|------|-----|---------------------------------|
| メモリ | 非共有（各ノード独立） | 共有（単一アドレス空間） |
| スケール方式 | 水平スケール（ノード追加） | 垂直スケール（CPU 追加） |
| データ分散 | ノード間でデータを分割 | 単一メモリ上のデータに全 CPU アクセス |
| 適用規模 | ペタバイト級 | テラバイト級まで |
| 障害耐性 | ノード単位で分離 | 共有リソースに依存 |

### データウェアハウスでの活用

MPP は [[データウェアハウス]] の大規模化に伴い，[[対称型マルチプロセシング]]（SMP）の後継として普及した．代表的な MPP 型 DWH:

| サービス | プラットフォーム | 特徴 |
|---------|----------------|------|
| Amazon Redshift | AWS | カラム指向，MPP，クラウドネイティブ |
| Google BigQuery | GCP | サーバーレス MPP，ストレージ/コンピュート分離 |
| Azure Synapse Analytics | Azure | SQL DW 後継，MPP + Spark |
| Snowflake | マルチクラウド | 仮想ウェアハウス，完全なコンピュート/ストレージ分離 |
| Databricks SQL | マルチクラウド | Photon エンジン，Lakehouse 統合 |

### スケーラビリティ

MPP の水平スケーラビリティは大規模データ処理に適している:
- ノードを追加すると，データと処理が自動的に再分散
- 追加コストが高価な大型サーバーではなく，汎用サーバーで賄える
- クラウドでは需要に応じた動的スケールアップ/ダウンが可能

### データ分散方式

- **ハッシュ分散**: 特定カラムのハッシュ値でノードに割り当て（結合キーに使用）
- **ランダム分散（ラウンドロビン）**: 均等分散，集計クエリに適する
- **複製（Replicate）**: 小テーブルを全ノードにコピー，大テーブルとの結合を高速化

## ポイント

- 大量データを扱う [[データウェアハウス]] の標準アーキテクチャ
- ノード間のデータシャッフルが発生するクエリ（ソート，結合）はボトルネックになりやすい
- クラウド時代はコンピュートとストレージの分離が進み，オンデマンドスケールが一般化
- Amazon Redshift，Google BigQuery などのクラウド DWH はすべて MPP ベース
- カラム指向ストレージと組み合わせることで集計クエリの性能が大幅向上
