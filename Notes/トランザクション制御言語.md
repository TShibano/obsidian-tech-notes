## 概要

TCL（Transaction Control Language）．トランザクションの開始・確定・取り消しを制御するための SQL の分類．データの整合性を保ちながら複数の DML 操作をアトミックに実行するために使用する．

## 詳細

### 主な TCL コマンド

| コマンド | 説明 |
|---|---|
| `BEGIN` / `START TRANSACTION` | トランザクションを開始 |
| `COMMIT` | トランザクションの変更を永続化 |
| `ROLLBACK` | トランザクションの変更を取り消し |
| `SAVEPOINT` | ロールバックポイントを設定 |
| `ROLLBACK TO SAVEPOINT` | 指定したセーブポイントまでロールバック |

### 使用例

```sql
BEGIN;

UPDATE accounts SET balance = balance - 1000 WHERE id = 1;
UPDATE accounts SET balance = balance + 1000 WHERE id = 2;

-- エラーが発生したらロールバック
COMMIT;  -- または ROLLBACK;
```

### SQL 言語分類における位置付け

| 分類 | 略称 | 主なコマンド |
|---|---|---|
| [[データ定義言語]] | DDL | CREATE, ALTER, DROP, TRUNCATE |
| [[データ操作言語]] | DML | SELECT, INSERT, UPDATE, DELETE |
| [[データ制御言語]] | DCL | GRANT, REVOKE |
| トランザクション制御言語 | TCL | BEGIN, COMMIT, ROLLBACK |

### ACID 特性との関係

TCL は [[ACID特性]] の実現手段：

- **A（Atomicity，原子性）**: COMMIT/ROLLBACK で全実行か全取り消しを保証
- **C（Consistency，一貫性）**: トランザクション前後でデータ整合性を維持
- **I（Isolation，独立性）**: 並行トランザクション間の干渉を防止
- **D（Durability，永続性）**: COMMIT 後のデータは障害があっても保持

### 注意点

- DDL（CREATE，DROP 等）は多くの DBMS で自動コミットのため ROLLBACK 不可
- 一部の DBMS では TCL を DCL の一部として分類する場合もある
- 分散トランザクション（複数 DB にまたがる）は[[アトミックトランザクション]]参照

## ポイント

- エラーハンドリングと組み合わせて，パイプラインの原子性を保証する
- 長いトランザクションはロック競合の原因になるため，最小限に保つ
- SAVEPOINT を使うと，一部だけロールバックするきめ細かい制御が可能
