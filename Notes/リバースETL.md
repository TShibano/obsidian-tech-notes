## 概要

リバースETL（Reverse ETL）は，データウェアハウスやデータレイクに蓄積・加工されたデータを，CRM・MA・広告プラットフォームなどの業務ツールに転送・同期するデータパイプラインパターンである．従来の ETL（ソース→DWH）の逆方向（DWH→業務ツール）のデータフローを実現し，分析データをオペレーショナルな業務プロセスで活用できるようにする．

## 詳細

### 従来ETL との比較

```
ETL（従来）:
業務システム → [Extract] → [Transform] → [Load] → DWH
                                              分析・BI へ

リバースETL:
DWH → [Extract] → [Transform] → [Load] → CRM / MA / 広告プラットフォーム
                                           業務プロセスへ
```

| 観点 | ETL | リバースETL |
|------|-----|------------|
| データの流れ | ソース → DWH | DWH → 業務ツール |
| 目的 | 分析・BI のためのデータ統合 | 分析データを業務活用 |
| 出力先 | DWH（Snowflake，BigQuery 等） | CRM，MA，広告，Slack 等 |
| ユーザー | データエンジニア，アナリスト | マーケター，営業，カスタマーサクセス |

### Operational Analytics（オペレーショナル分析）

リバースETLが実現する概念が「Operational Analytics」：分析データをリアルタイムに業務ツールに配信し，現場担当者がデータドリブンなアクションを取れるようにする．

例：
- DWH のユーザーセグメントをリアルタイムで広告プラットフォームに同期 → ターゲティング精度向上
- 顧客のライフタイムバリュー（LTV）を CRM に転送 → 営業優先順位付け
- 離脱リスクスコアをカスタマーサクセスツールに配信 → プロアクティブなフォローアップ

### 主要なリバースETLツール（2026）

| ツール | 特徴 |
|-------|------|
| **Hightouch** | リバースETL のリーディングカンパニー．200+ コネクタ |
| **Census** | SQL ベースの設定が特徴．データ品質チェック機能 |
| **Polytomic** | ノーコード対応．中小企業向け |
| **dbt + Hightouch** | dbt でデータ変換 → Hightouch で配信 の組み合わせが一般的 |

### アーキテクチャ上の位置づけ

```
業務システム群
    ↓ (ETL/ELT)
DWH / データレイク
    ↓ (変換・集計 by dbt 等)
データマート（分析済みデータ）
    ↓ (リバースETL)
CRM / MA / Slack / 広告プラットフォーム
    ↓
現場の業務アクション
```

## ポイント

- 「分析で終わらせず，業務にフィードバックする」のがリバースETLの本質
- ETL でデータを DWH に集め，リバースETL で業務ツールに配信する双方向のパイプラインが現代のデータ活用の標準形
- Hightouch が市場を牽引しており，dbt との組み合わせが定番構成
- Operational Analytics の実現に不可欠なコンポーネント
- マーケティング・営業・CS が自分でデータ活用できる「データの民主化」に貢献
