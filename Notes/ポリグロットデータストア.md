## 概要

ポリグロットデータストア（Polyglot Data Store）は，[[ポリグロット永続化]] の実践として用いられる，異なるデータモデルをサポートする多様なデータストアの総称である．また，単一のシステムが複数のデータモデル（リレーショナル・ドキュメント・グラフ等）を内包する「マルチモデルデータベース」を指す文脈でも用いられる．

## 詳細

### 2つの意味

1. **広義**: ポリグロット永続化において使用される，特性の異なる複数のデータストア群
2. **狭義**: 1つのDBMSが複数のデータモデルをサポートする「マルチモデルDB」（例: ArangoDB，Cosmos DB）

### データストアタイプと選定基準

| 種別           | データモデル           | 強み              | 弱み               | 選定基準                    |
| ------------ | ---------------- | --------------- | ---------------- | ----------------------- |
| **RDBMS**    | テーブル（行列）         | ACID，結合，複雑なクエリ  | スケールアウト困難，スキーマ硬直 | トランザクション必須，正規化データ       |
| **ドキュメントDB** | JSON/BSON ドキュメント | スキーマ柔軟，ネスト対応    | 複雑な結合が苦手         | 階層構造データ，スキーマが変化しやすい     |
| **KV ストア**   | キー→バリューペア        | 超高速読み書き，スケールアウト | クエリ表現力が低い        | キャッシュ，セッション，シンプルなルックアップ |
| **グラフDB**    | ノード＋エッジ          | 多段リレーション，グラフ探索  | 大量データの集計は苦手      | SNS関係，経路探索，知識グラフ        |
| **ワイドカラム型**  | カラムファミリー         | 大量書き込み，分散耐障害性   | 柔軟なクエリが難しい       | IoT，ログ，タイムライン           |
| **時系列DB**    | タイムスタンプ付き計測値     | 時刻順の効率的な格納・集計   | 汎用クエリは不得意        | 監視，センサー，金融データ           |
| **全文検索エンジン** | 転置インデックス         | 高速全文検索，スコアリング   | 更新コストが高い         | テキスト検索，ログ解析             |
| **ベクトルDB**   | 高次元ベクトル          | 類似ベクトル検索（ANN）   | 汎用クエリには不向き       | AI/ML，セマンティック検索         |

### マルチモデルデータベース

複数のデータモデルを単一エンジンでサポートする DB：

| 製品                  | サポートモデル            | 特徴              |
| ------------------- | ------------------ | --------------- |
| **ArangoDB**        | ドキュメント，グラフ，KV      | オープンソース，AQL クエリ |
| **Azure Cosmos DB** | ドキュメント，KV，グラフ，テーブル | フルマネージドマルチモデル   |
| **Couchbase**       | ドキュメント，KV，全文検索     | モバイル対応          |
| **OrientDB**        | ドキュメント，グラフ，オブジェクト  | Java ベース        |

マルチモデル DB を採用すると，複数の独立した DB の運用コストを削減しながら，ポリグロット永続化の一部のメリットを享受できる．

### eコマースでの適用例

```
商品カタログ → MongoDB（属性がカテゴリごとに異なる）
在庫・注文 → PostgreSQL（ACID トランザクション）
セッション・カート → Redis（高速 KV）
レコメンド → Neo4j（ユーザー×商品の関係グラフ）
商品検索 → Elasticsearch（全文検索＋ファセット）
クリックログ → Cassandra（時系列大量書き込み）
画像類似検索 → PGVector/Pinecone（ベクトル検索）
```

## ポイント

- データストアの選定は「データの形」「アクセスパターン」「スケール要件」の3軸で考える
- マルチモデル DB は運用コスト削減の手段だが，専用 DB に比べてパフォーマンスで劣る場合がある
- [[ポリグロット永続化]] の原則に従い，各サービスが最適なストアを持つことが基本
- ベクトル DB は LLM/AI ワークロードの増加とともに主要データストアの地位を確立しつつある
