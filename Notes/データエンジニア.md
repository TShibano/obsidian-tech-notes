## 概要

データエンジニア（Data Engineer）は，データの収集・変換・格納・配信に関するデータパイプラインとデータ基盤を設計・構築・運用する役割である．データ分析やAI/MLを支えるインフラを担当し，[[データアナリスト]] や [[データサイエンティスト]] がデータを活用できる環境を整備する．

## 詳細

### 主な職務

| 責務 | 内容 |
|------|------|
| **データパイプライン構築** | ETL/ELT パイプラインの設計・実装・テスト |
| **データ基盤整備** | DWH，データレイク，ストリーミング基盤の構築・維持 |
| **データ品質確保** | データのクレンジング，バリデーション，品質監視 |
| **ワークフロー管理** | Apache Airflow 等によるパイプラインのオーケストレーション |
| **パフォーマンス最適化** | クエリ最適化，パーティショニング，インデックス設計 |
| **セキュリティ・権限管理** | アクセス制御，暗号化，コンプライアンス対応 |

### 他のデータ役割との比較

| 観点 | データエンジニア | [[データアーキテクト]] | [[データアナリスト]] | [[データサイエンティスト]] |
|------|---------------|-----------------|-----------------|---------------------|
| 主な作業 | パイプライン構築・運用 | アーキテクチャ設計 | データ分析・可視化 | ML/統計モデリング |
| 主な成果物 | データパイプライン，インフラ | 設計書，標準 | レポート，ダッシュボード | 予測モデル，アルゴリズム |
| プログラミング | 必須（Python，Scala 等） | 設計中心（実装は補助的） | SQL，BI ツール中心 | Python，R，SQL |
| インフラ知識 | 必須 | 必要（設計レベル） | 不要 | 不要〜基本的なレベル |

### 主なスキルセット

- **プログラミング**: Python，Scala，Java（Spark 等の処理エンジン向け）
- **SQL**: 複雑なクエリ，パフォーマンスチューニング
- **データ基盤**: Snowflake，BigQuery，Redshift，Delta Lake
- **オーケストレーション**: Apache Airflow，Prefect，Dagster
- **変換**: dbt（データ変換の標準ツール）
- **ストリーミング**: Apache Kafka，Flink
- **クラウド**: AWS，GCP，Azure のデータサービス

### モダンデータスタックでの役割

```
データソース群（業務システム，API，ログ）
    ↓ [データエンジニアが構築]
ETL/ELT ツール（Fivetran，Airbyte 等）
    ↓
DWH（Snowflake，BigQuery 等）
    ↓ [データエンジニアが管理]
dbt（データ変換・モデリング）
    ↓
データマート
    ↓ [データアナリスト・データサイエンティストが活用]
BI ツール / ML モデル
```

## ポイント

- データエンジニアは「データを届けるインフラを作る人」であり，分析や ML の土台を支える
- [[ETL]]/[[ELT]] パイプラインの構築が中核業務
- 2026年のトレンド：dbt の普及，データ品質ツール（Great Expectations 等）の標準化，ストリーミング処理の増加
- [[データアーキテクト]] の設計を実装し，[[データアナリスト]] が使える環境を整備する役割
- モダンデータスタックの普及でエンジニアリング範囲が変化（ETL の一部が SaaS ツールに置換）

## 関連項目

- [[ETL]] — データエンジニアが構築する代表的なデータ取り込みパターン
- [[ELT]] — クラウド DWH 上での変換パターン
- [[データウェアハウス]] — データエンジニアが構築・管理する分析基盤
- [[データアーキテクト]] — データエンジニアの上位設計を担う役割
- [[データアナリスト]] — データエンジニアが整備した環境を活用して分析を行う役割
- [[データアーキテクチャ]] — データエンジニアが実装するデータ基盤の全体像
