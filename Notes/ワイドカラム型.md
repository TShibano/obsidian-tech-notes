## 概要

ワイドカラム型（Wide-Column Store / カラムファミリーストア）は，データをカラムファミリーと呼ばれる単位でグルーピングし，行ごとにカラムの数・名前が異なり得る NoSQL データベースである．大量の書き込みと水平スケーラビリティに優れており，IoT・ログ・SNS タイムラインなど時系列性のある大量データの格納に適する．

## 詳細

### データモデル

ワイドカラム型はリレーショナルのテーブルに見えるが，各行がそれぞれ異なるカラムを持てる：

```
行キー     | カラムファミリー: profile          | カラムファミリー: activity
---------- | -------------------------------- | ---------------------------
user:1001  | name:"田中" | email:"t@ex.com"   | last_login:2026-05-05 | posts:42
user:1002  | name:"佐藤"                       | last_login:2026-05-04 | views:1200 | premium:true
user:1003  | name:"鈴木" | phone:"090-xxxx"   |
```

各行は「行キー」で識別され，カラムは動的に追加できる（スパースなデータに効率的）．

### RDBMS・ドキュメントDBとの比較

| 観点 | ワイドカラム型 | RDBMS | ドキュメントDB |
|------|-------------|-------|-------------|
| スキーマ | カラムファミリーで管理（スパース） | 固定テーブル | スキーマレス |
| 書き込み | 超高スループット | 中程度 | 高い |
| 読み取り | キー or カラムファミリー指定 | 任意 SQL | ドキュメント全体 |
| JOIN | なし | あり | なし（ネスト） |
| スケール | 水平スケール（ペタバイト級） | 垂直中心 | 水平スケール |
| 一貫性 | 結果整合性（Cassandra）または強整合（HBase） | ACID | 制限的 ACID |

### 主要製品

| 製品 | 特徴 |
|------|------|
| **Apache Cassandra** | ピア・ツー・ピア分散．マスターなし．高可用性・結果整合性 |
| **Apache HBase** | Hadoop（HDFS）上に構築．強整合性．Google Bigtable の OSS 実装 |
| **Google Bigtable** | Google の独自実装．Cloud Bigtable としてクラウド提供 |
| **Amazon Keyspaces** | Cassandra 互換のフルマネージドサービス（AWS） |
| **ScyllaDB** | Cassandra 互換だが C++ で再実装．高性能 |

### Cassandra と HBase の違い

| 観点 | Cassandra | HBase |
|------|-----------|-------|
| アーキテクチャ | ピア・ツー・ピア（マスターなし） | マスター・スレーブ（HMaster） |
| 一貫性 | 結果整合性（Tunable） | 強整合性 |
| 基盤 | 独立分散ストレージ | HDFS |
| 書き込み速度 | 非常に高い | 高い |
| 読み取り速度 | 高い | 高い（HBase 最適化が必要） |

### ユースケース

- **SNS タイムライン**: ユーザーごとに大量の投稿を時系列で格納（Cassandra）
- **IoT センサーデータ**: デバイス ID をキーにタイムスタンプ付きの計測値を蓄積
- **ログ蓄積**: 大量ログのリアルタイム書き込みと時間範囲クエリ
- **広告エンジン**: ユーザープロファイルの超低レイテンシ読み書き
- **ゲームスコア**: プレイヤー×アイテムの大規模スパースデータ

## ポイント

- 書き込みスループットとペタバイト級の水平スケールが最大の強み
- キーと（カラムファミリー）を基準にアクセスパターンを設計することが重要（後からの変更が難しい）
- JOIN がないため，RDBMS のような任意の集計クエリには向かない
- [[ポリグロット永続化]] ではログ・タイムライン・IoT 蓄積の担当として採用されることが多い
- Apache Spark + Cassandra/HBase の組み合わせが大規模バッチ分析の定番スタック
