## 概要

関数型プログラミング（FP: Functional Programming）は、純粋関数・不変データ・副作用の排除を核とするプログラミングパラダイム。数学的な関数の概念に基づき「同じ入力→同じ出力」を保証し、状態管理の複雑さや予期せぬ副作用に起因するバグを根本から排除する。Haskell・Erlang が純粋 FP 言語で、Scala・Rust・Python など多くの言語が FP スタイルを取り込んでいる。

## 詳細

### 核心概念

#### 純粋関数（Pure Function）

同じ入力に対して常に同じ出力を返し、外部の状態を変更しない関数。

```python
# 純粋関数（推奨）
def add(a: int, b: int) -> int:
    return a + b  # 同じ引数なら常に同じ結果

# 純粋でない関数（副作用あり）
total = 0
def add_to_total(x: int) -> int:
    global total
    total += x  # 外部状態を変更→副作用
    return total
```

**メリット:** テストが容易（モックが不要）、並列実行が安全、推論しやすい

#### 不変性（Immutability）

一度作ったデータを変更しない。変更が必要なら新しいデータを作る。

```python
# ミュータブル（OOP 的）
numbers = [1, 2, 3]
numbers.append(4)  # 元のリストを破壊的に変更

# イミュータブル（FP 的）
numbers = (1, 2, 3)  # タプル（変更不可）
new_numbers = numbers + (4,)  # 新しいタプルを作成
```

Rust は `let` でイミュータブルがデフォルト、`let mut` で明示的にミュータブルを選ぶ。

#### 参照透過性（Referential Transparency）

式をその評価結果で置き換えてもプログラムの振る舞いが変わらない性質。純粋関数を使えば自動的に達成される。

```python
# 参照透過な式
result = add(2, 3)  # → 5
# add(2, 3) を 5 に置き換えても同じ動作
```

### 高階関数（Higher-Order Functions）

関数を引数として受け取ったり、関数を返したりする関数。

```python
# map: 各要素に関数を適用
numbers = [1, 2, 3, 4, 5]
squares = list(map(lambda x: x**2, numbers))
# [1, 4, 9, 16, 25]

# filter: 条件を満たす要素を選択
evens = list(filter(lambda x: x % 2 == 0, numbers))
# [2, 4]

# reduce: 畳み込み
from functools import reduce
total = reduce(lambda acc, x: acc + x, numbers, 0)
# 15
```

### 関数合成（Function Composition）

小さな純粋関数を組み合わせて複雑な処理を構築する。

```python
# 関数合成
def compose(*fns):
    def composed(x):
        for fn in reversed(fns):
            x = fn(x)
        return x
    return composed

double = lambda x: x * 2
add_one = lambda x: x + 1
square = lambda x: x ** 2

transform = compose(square, add_one, double)
print(transform(3))  # square(add_one(double(3))) = square(7) = 49
```

### クロージャ（Closure）

関数が定義されたスコープの変数を「閉じ込める」仕組み。副作用なしに状態のような概念を実現する。

```python
def make_counter(start: int = 0):
    count = [start]  # リストで包んでイミュータブルにする代わりにクロージャで管理
    def increment():
        count[0] += 1
        return count[0]
    return increment

counter = make_counter(10)
print(counter())  # 11
print(counter())  # 12
```

### モナド（Monad）

副作用・エラー・非同期・オプショナル値などを型安全に扱う抽象化パターン。

| モナド | 用途 | 言語例 |
|--------|------|--------|
| `Maybe` / `Option` | null 安全 | Haskell / Rust |
| `Result` / `Either` | エラーハンドリング | Rust / Haskell |
| `IO` | 副作用の隔離 | Haskell |
| `Promise` / `Future` | 非同期処理 | JavaScript / Scala |
| `List` | 非決定的計算 | Haskell |

```rust
// Rust の Option モナド（Haskell の Maybe に相当）
fn find_user(id: u32) -> Option<String> {
    if id == 1 { Some("Alice".to_string()) } else { None }
}

// and_then で連鎖（モナディックチェイン）
let result = find_user(1)
    .and_then(|name| if name.len() > 3 { Some(name) } else { None })
    .unwrap_or("Unknown".to_string());
```

### 遅延評価（Lazy Evaluation）

値が実際に必要になるまで評価を遅らせる。無限リストなどを扱えるようになる。

```python
# Python のジェネレータは遅延評価
def infinite_naturals():
    n = 0
    while True:
        yield n
        n += 1

gen = infinite_naturals()
print(next(gen))  # 0
print(next(gen))  # 1
```

### 主要な FP 言語

| 言語 | FP の程度 | 特徴 |
|------|----------|------|
| Haskell | 純粋 FP | 静的型付け、遅延評価、モナド重視 |
| Erlang / Elixir | 純粋 FP 寄り | 並行処理・耐障害性に特化 |
| Clojure | 実用的 FP | JVM 上の Lisp、イミュータブルデータ構造 |
| Scala | FP + OOP | JVM、Cats/ZIO などのエコシステム |
| F# | FP 中心 | .NET、型推論、パターンマッチ |
| [[Rust]] | FP 取り込み | 所有権 + イテレータ + Option/Result |
| Python | マルチパラダイム | map/filter/lambda + functools |
| JavaScript | マルチパラダイム | 高階関数 + クロージャ + Ramda 等 |

### [[Nix]] との関係

[[Nix]] パッケージマネージャーは**純関数型の設計原則**を OS のパッケージ管理に適用したもの。同じ入力（`default.nix`）から常に同じ出力（ビルド結果）を保証することで再現性を実現している。Nix 言語自体も純関数型言語。

### OOP との比較

| 観点 | FP | [[オブジェクト指向プログラミング]] |
|------|----|--------------------|
| 状態管理 | 不変（イミュータブル）| ミュータブルな状態を持つ |
| 副作用 | 排除・隔離 | 許容 |
| 並列処理 | 安全（共有状態なし） | 複雑（ロック必要） |
| テスト | 容易（純粋関数） | モック必要 |
| 学習曲線 | 急（圏論・モナド等） | 緩やか |
| 適する領域 | データ変換・並行処理 | 状態を持つビジネスロジック |

## ポイント

- 純粋関数 + 不変データで「状態に起因するバグ」を根本から防ぐ
- 高階関数（map/filter/reduce）と関数合成で簡潔・再利用可能なコードを書く
- モナドはエラー・null・副作用を型安全に扱うための抽象化
- Rust は FP の概念（イミュータブル・Option・Result・イテレータ）を実用的な形で取り込んでいる
- Nix は FP の「同じ入力→同じ出力」をパッケージ管理に適用した例
