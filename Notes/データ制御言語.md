## 概要

DCL（Data Control Language）．データベースへのアクセス権限を制御するための SQL の分類．ユーザやロールに対して権限の付与・剥奪を行い，データのセキュリティとガバナンスを実現する．

## 詳細

### 主な DCL コマンド

| コマンド | 説明 |
|---|---|
| `GRANT` | 権限を付与する |
| `REVOKE` | 権限を剥奪する |

### 使用例

```sql
-- ユーザ tanaka に users テーブルへの SELECT 権限を付与
GRANT SELECT ON users TO tanaka;

-- 全テーブルへの全権限を付与
GRANT ALL PRIVILEGES ON ALL TABLES TO admin_user;

-- 権限を剥奪
REVOKE SELECT ON users FROM tanaka;

-- ロールを使った権限管理（PostgreSQL の例）
CREATE ROLE analyst;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO analyst;
GRANT analyst TO tanaka;
```

### SQL 言語分類における位置付け

| 分類 | 略称 | 主なコマンド |
|---|---|---|
| [[データ定義言語]] | DDL | CREATE, ALTER, DROP, TRUNCATE |
| [[データ操作言語]] | DML | SELECT, INSERT, UPDATE, DELETE |
| データ制御言語 | DCL | GRANT, REVOKE |
| [[トランザクション制御言語]] | TCL | BEGIN, COMMIT, ROLLBACK |

### データエンジニアリングでの役割

- **最小権限の原則**: パイプラインのサービスアカウントに必要最小限の権限のみ付与
- **行レベルセキュリティ**: 特定行への閲覧制限（BigQuery，Snowflake 等でサポート）
- **列レベルセキュリティ**: 個人情報含む列へのアクセス制限
- **ロールベースアクセス制御 (RBAC)**: ロールで権限をまとめて管理

### データガバナンスとの関係

DCL はデータガバナンスの技術的実現手段．[[データカタログ]] やポリシー管理ツールと組み合わせて，組織全体のアクセス制御ポリシーを実装する．

## ポイント

- `GRANT` と `REVOKE` は即座に反映（DDL と同様に自動コミット）
- ロールを活用することで，ユーザ単位の権限管理を簡素化できる
- クラウドデータウェアハウス（BigQuery，Snowflake）では IAM と統合した権限管理が主流
