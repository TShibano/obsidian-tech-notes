# デイリーニュース作成

以下の手順でデイリーニュースを作成してください。

## 1. 日付の決定

- `$ARGUMENTS` が指定されている場合、その値を日付（YYYY-MM-DD 形式）として使用する
- 指定がない場合は今日の日付を使用する

## 2. 既存ファイルの確認

- `DailyNews/YYYY-MM-DD.md` が既に存在するか確認する
- 存在する場合はユーザーに上書きするか確認を取る

## 3. ニュースの収集

CLAUDE.md の「ニュースソース」セクションに定義されたソースから、WebSearch を使って当日の技術ニュースを収集する。

## 4. フィルタリングと優先順位

以下の優先順位でニュースを選別する:

1. **AI / ML** 関連（最優先）
2. **Web / Cloud** 関連
3. **その他の注目ニュース**

## 5. arXiv 論文の収集

WebSearch で arXiv の最新論文を検索し、注目論文を3〜5本ピックアップする。

- 対象カテゴリ: `cs.AI`, `cs.CL`, `cs.CV`, `cs.LG`, `cs.MA` 等
- 直近数日以内に投稿・更新された論文を対象とする
- 選別基準: 新手法の提案、SOTA 更新、実用的応用（コード公開あり優先）、SNS等で話題の研究
- 各論文は日本語で1行に要約し、arXiv の abs ページへリンクする
- 著者欄は「筆頭著者 et al.」の形式で記載する

## 6. ノートの作成

- `Templates/daily-news.md` のテンプレート構造に従って記述する
- 各記事・論文は日本語で1行に要約する
- frontmatter の `date` に対象日付を設定する
- 記事や論文がないカテゴリのテーブルにはデータ行を含めない
- **URLは必ず一次情報源を使用する**（公式ブログ、公式発表、論文、プレスリリースなど）。まとめ記事やニュースアグリゲーターのURLは使わない。WebSearch の結果がまとめ記事経由の場合は、追加検索で一次情報源を特定する

## 7. ファイルの保存

- `DailyNews/YYYY-MM-DD.md` として保存する

## 8. コミットとプッシュ

作成したデイリーニュースファイルをコミットしてリモートにプッシュする:

1. 対象ファイル（`DailyNews/YYYY-MM-DD.md`）を `git add` でステージング
2. 以下の形式でコミットメッセージを作成し `git commit` する:
   - 形式: `デイリーニュース(YYYY/MM/DD): 主要トピック1, トピック2, ...等`
   - 例: `デイリーニュース(2026/02/10): OpenAI Frontier, Big Tech AI投資$650B, Agent Trace等`
   - AI/ML カテゴリから主要な2〜3件をピックアップしてコミットメッセージに含める
3. `git push` でリモートにプッシュ
